1. Pendahuluan &amp; Latar Belakang
1.1 Deskripsi Produk
Sistem Digital Setoran Tahfidz (SITA)
 adalah aplikasi manajemen hafalan Al-Qur'an berbasis web yang dirancang untuk mendigitalisasi proses 
talaqqi
 (setoran hafalan secara langsung antara santri dan ustadz). Sistem ini memungkinkan ustadz/pengajar untuk menyimak hafalan, menandai kesalahan pada tingkat kata (
word-level bookmarking
), dan memberikan penilaian secara otomatis dan objektif.
1.2 Latar Belakang Masalah
Proses setoran hafalan Al-Qur'an (talaqqi) secara tradisional memiliki beberapa kelemahan:
Masalah
Dampak
Catatan kesalahan ditulis manual di kartu/buku
Rawan hilang, sulit dianalisis secara historis
Penilaian bersifat subjektif per ustadz
Standar kelulusan tidak konsisten
Orang tua tidak memiliki akses real-time
Sulit memantau perkembangan anak
Tidak ada data agregat kesalahan
Santri tidak tahu pola kesalahan berulang
Rekap laporan dilakukan manual
Membutuhkan waktu admin yang signifikan
1.3 Solusi yang Ditawarkan
SITA hadir sebagai platform digital yang menjembatani seluruh stakeholder dalam ekosistem tahfidz: ustadz, santri, orang tua, dan manajemen lembaga — dengan satu sistem terintegrasi berbasis data.
2. Tujuan &amp; Sasaran Produk
2.1 Tujuan Utama
Digitalisasi Proses Talaqqi
 — Menggantikan kartu setoran fisik dengan sistem digital yang akurat dan real-time.
Presisi Feedback
 — Memberikan informasi spesifik hingga level kata/huruf yang salah kepada santri.
Objektivitas Penilaian
 — Mengotomasi kalkulasi nilai berdasarkan aturan baku yang dapat dikonfigurasi lembaga.
Transparansi Progres
 — Menyediakan akses data hafalan bagi santri dan orang tua secara real-time.
Analitik Lembaga
 — Menghasilkan laporan agregat untuk manajemen mutu hafalan di tingkat lembaga.

5. Fitur Utama — Detail Lengkap
5A. Modul Autentikasi &amp; Manajemen Akun
5A.1 Login &amp; Registrasi
Login menggunakan username/email dan password.
Admin mendaftarkan akun ustadz dan santri secara bulk (via import CSV).
Santri dapat didaftarkan manual oleh admin dengan pengisian data: nama lengkap, NIS, tanggal lahir, kelas/halaqah, wali.
Dukungan lupa password via email reset link.
5A.2 Manajemen Profil
Setiap pengguna dapat mengedit data profil: foto, nomor kontak.
Admin dapat menonaktifkan/mengaktifkan akun tanpa menghapus data historis.
Tautan akun santri ke akun orang tua (relasi 
student_id
 → 
parent_id
).
5A.3 Struktur Halaqah / Kelompok
Admin membuat grup halaqah (misal: &quot;Kelas Tahfidz A&quot;, &quot;Halaqah Subuh&quot;).
Satu halaqah memiliki satu ustadz utama (dapat lebih dari satu sebagai musyrif).
Santri ditugaskan ke satu atau lebih halaqah.

5C. Interactive Quran Reader &amp; Word-Level Bookmarking
Ini adalah fitur inti SITA. Tampilan ini menjadi tempat sesi setoran berlangsung secara interaktif.
5C.1 Rendering Teks Al-Qur'an
Teks Al-Qur'an ditampilkan 
per kata
, bukan per baris biasa.
Setiap kata dirender sebagai elemen DOM yang dapat diklik/diketuk secara individual.
Font: 
Uthmanic Hafs
 (KFGQPC Uthmanic Script HAFS) atau IndoPak — dikonfigurasi via admin untuk konsistensi 
word_index
.
Arah teks: 
RTL (Right-to-Left)
 dengan tata letak yang mengikuti standar mushaf.
Nomor ayat ditampilkan di antara setiap ayat.
Ayat yang sedang &quot;aktif&quot; (sedang dibaca) dapat di-highlight secara manual oleh ustadz.

5D.2 Kalkulasi Skor
Skor Akhir = Max(0, Nilai Awal − (ΣJali × Penalty_Jali) − (ΣKhafi × Penalty_Khafi) − (ΣTark × Penalty_Tark))
Contoh:
Nilai awal: 100
2 Lahn Jali: −6
3 Lahn Khafi: −3
1 Tark: −2
Skor Akhir: 100 − 6 − 3 − 2 = 89 → Status: LULUS ✅

5D.4 Presisi Kalkulasi
Semua operasi aritmatika menggunakan bilangan bulat (integer) untuk menghindari floating-point error.
Jika di masa depan dibutuhkan poin desimal (misal −0.5), gunakan integer dengan skala 10x (simpan 995 untuk representasi 99.5).

5G. Dashboard Admin / Lembaga
5G.1 Manajemen Pengguna
CRUD (Create, Read, Update, Deactivate) untuk akun: Admin, Ustadz, Santri, Orang Tua.
Import data santri massal via file CSV dengan template yang disediakan.

Atur parameter penilaian global: nilai awal, penalty per kategori, threshold lulus.
Catat log perubahan konfigurasi (siapa yang mengubah, kapan, nilai sebelum-sesudah).
5G.3 Laporan &amp; Analitik Lembaga
Laporan Rekap Harian/Mingguan/Bulanan:
 Total sesi, rata-rata skor, persentase kelulusan.
Peringkat Santri:
 Berdasarkan total ayat lulus, rata-rata skor, konsistensi setoran.
Laporan Per Ustadz:
 Jumlah sesi disimak, distribusi nilai yang diberikan.
Analisis Kata Sulit:
 Kata-kata yang paling banyak salah di seluruh santri lembaga.
Ekspor laporan ke format PDF dan Excel.

7. Alur Pengguna (User Flow)
7.1 Flow Utama: Sesi Setoran (Ustadz)
[1] Login sebagai Ustadz
    │
[2] Dashboard Ustadz → Klik &quot;Mulai Sesi Baru&quot;
    │
[3] Form Inisiasi Sesi:
    - Pilih Santri (dari halaqah saya)
    - Pilih Jenis Setoran (Baru / Murajaah)
    - Pilih Surah → Ayat Mulai → Ayat Selesai
    - [Konfirmasi]
    │
[4] Interactive Quran Reader tampil
    - Teks Al-Qur'an dirender per kata (RTL)
    - Panel kontrol: Skor Live (100), tombol Selesai, Batal
    │

    - Ustadz klik/tap kata yang salah

9. Persyaratan Non-Fungsional
9.1 Performa
Metrik
Target
Waktu load halaman utama
&lt; 2 detik (3G)
Waktu render teks Al-Qur'an (10 ayat)
&lt; 1.5 detik
Response API endpoint utama
&lt; 500ms
Waktu simpan sesi (submit)
&lt; 3 detik
Concurrent users yang didukung
≥ 100 sesi aktif bersamaan
9.2 Keamanan
Semua koneksi menggunakan HTTPS (TLS 1.2+).
Password di-hash dengan bcrypt (cost factor ≥ 12).
JWT Access Token masa aktif 15 menit; Refresh Token 7 hari.
Validasi input di sisi server (tidak hanya client-side).
Authorization: setiap endpoint memvalidasi hak akses role pengguna.
Rate limiting pada endpoint login: max 5 percobaan gagal per menit.
Data orang tua hanya bisa melihat data anak yang terhubung (row-level security).
9.3 Keandalan &amp; Ketersediaan
Target uptime: 99.5% per bulan (downtime maksimal ~3.6 jam/bulan).
Database backup otomatis harian.
Graceful error handling: jika API gagal saat sesi aktif, state bookmark tersimpan di localStorage sebagai fallback sementara.
9.4 Aksesibilitas &amp; Kompatibilitas
Browser yang didukung: Chrome 90+, Firefox 88+, Safari 14+, Edge 90+.
Responsive design: desktop, tablet, dan mobile (minimal 375px lebar layar).
Sesi setoran dioptimalkan untuk penggunaan di tablet (ustadz umumnya menggunakan tablet saat mengajar).
Mendukung mode offline parsial: teks Al-Qur'an ter-cache untuk surah yang pernah diakses (via Service Worker).

12. Asumsi &amp; Batasan
12.1 Asumsi
Sumber data Al-Qur'an menggunakan 
Abdullah Ghanem Quran DB
 yang sudah tersegmentasi per kata dengan 
word_index
 konsisten.
Proses setoran dilakukan secara 
tatap muka
 (Ustadz dan Santri berada di lokasi sama); sistem tidak dirancang untuk setoran jarak jauh di v1.0.
Satu sesi hanya mencakup 
satu surah
 (tidak lintas surah).
Setiap ustadz hanya dapat mengelola sesi untuk santri di 
halaqah miliknya
.
Lembaga menggunakan satu konfigurasi penilaian yang sama untuk semua santri.
12.2 Batasan
Tidak ada fitur rekaman audio di v1.0.
Sistem tidak mendeteksi kesalahan secara otomatis; deteksi sepenuhnya dilakukan oleh ustadz.
Aplikasi mobile native tidak tersedia di v1.0; hanya web responsive.
Tidak ada integrasi eksternal (keuangan, absensi) di v1.0.
